###　更换不同api中转站配置

最后总结出来的配置三件套(因为我一直换各个不同的api中转站hhh)

（适用于windows powershell）

  1. 修改环境变量（这个好像是单进程生效）
$env:ANTHROPIC_API_KEY = "your_api_key"
$env:ANTHROPIC_BASE_URL = "中转站网址，大部分情况是网址，一般中转站有文档会写出"

检查是否成功输入（不要多括号和引号什么的）

echo $env:ANTHROPIC_API_KEY

echo $env:ANTHROPIC_BASE_URL

notice: don’t use both api_key and token，他们本质上作用是一样的，因为我看了不同的教程用了不同的东西导致我两个都设置了，然后就报错了，，

  1. 改长期配置（其实我不知道这个必不必要，但是在解决报错的过程中ai叫我这么做hhh）
notepad $PROFILE

创建新的文件是正常现象，然后把上一步的两行环境变量粘进去

  1. 改~/.claude/setting.json

有的教程又叫我用config,,,乱七八糟的

粘贴一些基础配置（同样我也不知道必不必要hhh，ai说的）

###　修复错误的简单历程

  1. 似乎是没有成功设置url，claude一直走官网拉取，报网络问题：您所在国家不能使用，开代理也不行 就是这一步诞生了上面一堆修复的尝试hhh，但是其实都没奏效， 卡了很久之后选择换教程，换了两三个教程看找到了上面贴的那个教程，通过一个windows power shell脚本绕开了初始的新手教程，拉取和配置，终于成功连上了

  2. 连上了之后开始报503错误：account not exist,问ai半天后选择去找客服，客服不理我

  3. 尝试换api中转站，找到这篇博客

换了api易之后，进去是进去了，但是开始报model not support，换了几个分组都不行

  1. 上一个中转站客服终于理我了，说是那个分组的号最近被封的厉害让我换个分组，我照做终于好了

省钱tips

  • 多用 claude ignore：把不需要扫描的文件夹（如 node_modules, dist）排除掉，减少 Context 消耗。

  • 控制对话深度：定时输入 /clear 或者重新开启对话，防止上下文堆叠导致单次请求 Token 爆炸。

  • 别跟风加一堆rules，那些也烧token
